# Demo Script

## Goal
Demonstrate lead-engineer thinking through a polished, production-minded clinician workflow.

## Demo flow

### 1. Open dashboard
Show:
- localized UI
- theme-aware UI
- KPI cards
- ranked patient roster
- sync/risk visibility

Narration:
“This dashboard is designed for Monday morning triage, not passive analytics.”

### 2. Open a high-priority patient
Show:
- summary header
- recent trend charts
- key supporting context
- alert/task panel
- explainable risk reasons

Narration:
“The important part is not the score alone, but why the patient is being surfaced.”

### 3. Generate visit prep summary
Show:
- traceable summary structure
- trends
- discussion points
- persisted generated summary

Narration:
“The summary is grounded in structured facts and intended to support review, not replace clinical judgment.”

### 4. Show audit trail
Show:
- sensitive actions and reads
- access traceability

Narration:
“This product is designed with healthcare-grade operational thinking.”

### 5. Show architecture briefly
Show:
- modular-monolith approach
- ingestion pipeline
- derived metrics
- risk engine
- observability mindset

Narration:
“I built this to show platform ownership, not just feature implementation.”
